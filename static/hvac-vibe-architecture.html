KEY DESIGN DECISIONS
On-device FFT
1024-pt FFT in ~2.4ms via CMSIS-DSP FPU. Physics-based features only — no ML on device. FFT computation is 20× faster with hardware FPU enabled.
Thread mesh transport
802.15.4 Thread replaces BLE. XIAO nRF52840 joins Pi border router as child node. CoAP NON POST to ff03::1:5683 (realm-local multicast). Sensor joins automatically using hardcoded dataset credentials.
RPM from FFT
1× harmonic peak search within ±10% of nameplate RPM. Quadratic interpolation gives sub-bin accuracy (~±12 RPM). No Hall sensor required in field.
Stats stay on Pi
Trending, anomaly detection, ML inference live on Pi and cloud. nRF computes physics — Pi does statistics. Clean separation of concerns.
FSM architecture preserved
Table-driven FSM in fsm.c gains one new PROCESSING state between SAMPLING and NOTIFYING. Semaphore handoff from DMA ISR to FSM thread.
Lock-free ring buffer
Single-producer (DMA) single-consumer (FSM) pattern. No mutex needed — atomic pointer advances with memory barriers. Power-of-2 sizing for fast modulo.

PROCESSING PIPELINE — PER ANALYSIS CYCLE (60s default)
01
DMA sample collection 640 ms
nRF SPI DMA pulls 1024 samples per axis from ADXL343 FIFO at 1600 Hz. Zero CPU involvement during capture. Lock-free ring buffer with power-of-2 sizing. FIFO interrupt triggers next DMA transfer. Semaphore posted to FSM when buffer half is full.
// DMA ISR — lock-free push buffer[write_ptr] = sample; __DMB(); write_ptr = (write_ptr + 1) & MASK; k_sem_give(&half_ready);
02
Convert + Hanning window ~0.8 ms
Convert int16 ADC counts to float32 g-values using scale factor. Apply Hanning window to reduce spectral leakage — critical for bearing fault frequency isolation. Window coefficients stored as const array in flash.
for (int i = 0; i < N; i++) { float w = 0.5f * (1.0f - arm_cos_f32(2*PI*i/N)); x[i] = (raw[i] * SCALE_G) * w; }
03
CMSIS-DSP FFT ~2.4 ms
arm_rfft_fast_f32() on 1024 points. Leverages FPU SIMD — 20× faster than software float. Output is complex spectrum. arm_cmplx_mag_f32() computes magnitude. Frequency resolution = 1600 / 1024 = 1.5625 Hz/bin.
arm_rfft_fast_instance_f32 fft; arm_rfft_fast_init_f32(&fft, 1024); arm_rfft_fast_f32(&fft, x, fft_out, 0); arm_cmplx_mag_f32(fft_out, mag, 512);
04
RPM estimation ~0.2 ms
Search FFT magnitude within ±10% of nameplate RPM. Find peak bin k. Apply quadratic interpolation using neighbors k±1 for sub-bin accuracy. Achieves ~±12 RPM precision. No Hall sensor needed in field deployment.
// Quadratic interpolation for sub-bin accuracy float delta = 0.5f * (mag[k-1] - mag[k+1]) / (mag[k-1] - 2.0f*mag[k] + mag[k+1]); float true_freq = (k + delta) * freq_resolution; uint16_t rpm = (uint16_t)(true_freq * 60.0f);
05
Bearing band energies ~0.3 ms
Compute BPFO, BPFI, BSF, FTF from RPM + bearing geometry constants (loaded at commissioning). Sum FFT magnitude in ±2-bin window around each fault frequency and harmonics. Computed per axis independently.
// BPFO for 6202 bearing (N_BALLS=9) float bpfo_hz = 0.4f * N_BALLS * rpm / 60.0f; int bin = (int)(bpfo_hz / freq_resolution); float energy = 0; for (int i = bin-2; i <= bin+2; i++) energy += mag[i];
06
Time domain features ~0.4 ms
RMS (overall energy), peak, crest factor (peak/RMS — impulsive sensitivity), kurtosis (4th moment of 1024-sample window — early fault indicator). All computed per axis using CMSIS-DSP primitives on FPU. Single-window statistics — no history needed.
arm_rms_f32(x, N, &rms); arm_abs_max_f32(x, N, &peak, &peak_idx); float crest = peak / rms; // Kurtosis: 4th moment / variance^2 float kurtosis = moment4 / (rms * rms * rms * rms);

THREAD FIT
104–112 bytes fits in a single 802.15.4 frame (127 byte max payload). CoAP NON POST — fire and forget, no ACK overhead.
FUTURE LORA FIT
104 bytes at SF9 BW125 ≈ 370ms airtime. At 60s interval = 0.62% duty cycle. Well within 1% regional limit. Same payload — only transport changes.
INT16 ENCODING
All fields encoded as integer with scaling (×100, ×10, raw counts). Avoids float on wire — smaller, alignment-safe, endian-safe on both nRF and Pi.
