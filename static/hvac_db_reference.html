engine.db — Database Reference

SQLite · WAL mode · /home/mitkatch/hvac-engine/engine.db

Overview

Three tables, two independent write paths, no foreign key constraints (SQLite pragmatics). Data flows from Thread sensor → engine → DB → MQTT → dashboard.

TablePurposeWrite rateRows/dayRetention
sensors One row per sensor, live state snapshot Every burst (~10s) 1 (upsert) Forever (manual)
history_time Per-minute time-domain aggregates Once per minute 1,440 max Manual / cron
history_fft Per-burst FFT fault energies Every burst (~10s) ~8,640 Manual / cron
Storage note: history_fft grows ~8,640 rows/day per sensor. At current count of 17,408 rows that's ~2 days of data. On a 32 GB SD card this is fine indefinitely, but a retention policy is recommended to keep query performance sharp.

history_time table

Per-minute time-domain features. Primary key is (sensor_id, date, minute_of_day) — within one minute, repeated bursts overwrite the row (last-write wins). Used by the dashboard VIB RMS chart.

Indexes

-- Fast chart queries (date range per sensor)
idx_time_sensor_date  ON history_time(sensor_id, date)
-- Cloud sync batch (find unsynced rows)
idx_time_unsynced     ON history_time(synced, date)

history_fft table

Append-only per-burst FFT bearing fault features. Unlike history_time, every burst creates a new row — this is the high-resolution fault detection log. minute_of_day allows JOIN with history_time for ML datasets.

x_snr_bpfoREALSNR at BPFO bin — >2.0 = outer race suspect
max_snr_bpfoREALMax SNR across all 3 axes

ML JOIN query

SELECT t.date, t.minute_of_day,
       t.vib_rms, t.kurtosis_x, t.crest_x,
       t.temp_c, t.alarm,
       f.x_bpfo, f.x_bpfi, f.x_snr_bpfo
FROM   history_time t
LEFT JOIN history_fft f
  ON  t.sensor_id     = f.sensor_id
  AND t.date          = f.date
  AND t.minute_of_day = f.minute_of_day
WHERE  t.sensor_id = 'hvac-vibe-2be618'
  AND  t.date BETWEEN '2026-06-01' AND '2026-06-04'
ORDER BY t.date, t.minute_of_day;

Write paths

Time-domain path (every burst ~10s)

XIAO Thread sensor → engine_thread.py
_parse_time_stats() → SensorSession
._publish_time_stats() → store.update_time_stats() → sensors (upsert)
history_time (upsert/min) → MQTT vibration/time_stats

FFT path (every burst ~10s)

XIAO Thread sensor → engine_thread.py
_parse_fft_stats() → SensorSession
._publish_fft_stats() → store.update_fft_stats() → history_fft (INSERT) → MQTT vibration/fft_stats
The two paths are completely independent. history_time uses INSERT OR REPLACE on (sensor_id, date, minute_of_day) — multiple bursts in the same minute overwrite. history_fft always appends a new row per burst.

Retention policy

There is currently no automatic deletion in engine_store.py. Data accumulates until manually purged. Recommended cron-based retention:

TableRecommended retentionReason
history_time90 daysPer-minute data — 90 days = 130K rows/sensor
history_fft30 daysPer-burst — grows 8,640 rows/day/sensor

Manual purge (run any time)

-- Keep last 90 days of per-minute data
DELETE FROM history_time
WHERE date < date('now', '-90 days');

-- Keep last 30 days of FFT data
DELETE FROM history_fft
WHERE date < date('now', '-30 days');

-- Reclaim space after bulk delete
VACUUM;

Install as nightly cron (2:00 AM)

# On Pi — run: crontab -e
0 2 * * * sqlite3 /home/mitkatch/hvac-engine/engine.db \
  "DELETE FROM history_time WHERE date < date('now','-90 days'); \
   DELETE FROM history_fft  WHERE date < date('now','-30 days'); \
   VACUUM;"

Add a new sensor

Step 1 — Flash the firmware with matching Thread credentials (channel 23, PAN 0xC442, network key). The sensor auto-joins.

Step 2 — Find the sensor's Thread IPv6 address on the Pi:

sudo ot-ctl child table
# Note the new child's RLOC16, then:
sudo ot-ctl ipaddr

Or watch engine logs for the auto-detected address:

sudo journalctl -u hvac-engine -f | grep "New Thread sensor"
# Output: New Thread sensor: thread-XXXX @ fd54:b06a:...

Step 3 — Map the address to a stable sensor_id in config.json:

{
  "thread_sensors": {
    "fd54:b06a:fe40:1:603b:9690:f851:ec4d": "hvac-vibe-2be618",
    "fd54:b06a:fe40:1:XXXX:XXXX:XXXX:XXXX": "hvac-vibe-unit2"
  }
}

Step 4 — Restart the engine:

sudo systemctl restart hvac-engine

Step 5 — Verify the sensor appears in the DB:

sqlite3 ~/hvac-engine/engine.db \
  "SELECT sensor_id, name, last_seen FROM sensors;"

Remove / retire a sensor

Removing a sensor from config.json stops new data from being written but does NOT delete historical data. Delete history explicitly if needed.

Step 1 — Remove from config.json (delete the thread_sensors entry).

Step 2 — Restart the engine.

Step 3 — Optionally delete all historical data:

-- Replace 'old-sensor-id' with the actual sensor_id
sqlite3 ~/hvac-engine/engine.db "
DELETE FROM sensors      WHERE sensor_id='old-sensor-id';
DELETE FROM history_time WHERE sensor_id='old-sensor-id';
DELETE FROM history_fft  WHERE sensor_id='old-sensor-id';
VACUUM;
"

To keep history but hide from dashboard — just remove from config.json. The rows remain for ML/analysis use.

Purge stale / ghost sensors

After migrating from BLE to Thread, ghost sensor IDs (e.g. thread-ec4d) may exist. Clean them up:

-- List all sensors with last_seen timestamp
sqlite3 ~/hvac-engine/engine.db \
  "SELECT sensor_id, name,
          datetime(last_seen,'unixepoch','localtime') as last_seen
   FROM sensors ORDER BY last_seen DESC;"

-- Delete a ghost sensor and all its history
sqlite3 ~/hvac-engine/engine.db \
  "DELETE FROM sensors      WHERE sensor_id='thread-ec4d';
   DELETE FROM history_time WHERE sensor_id='thread-ec4d';
   DELETE FROM history_fft  WHERE sensor_id='thread-ec4d';
   VACUUM;"

Backup

SQLite WAL mode is safe to copy while the engine is running using the .backup command:

# Safe online backup (no engine stop needed)
sqlite3 ~/hvac-engine/engine.db \
  ".backup /home/mitkatch/hvac-engine/engine.db.bak"

# Or copy to another location with timestamp
sqlite3 ~/hvac-engine/engine.db \
  ".backup /tmp/engine-$(date +%Y%m%d).db"

Automated weekly backup via cron

# crontab -e — runs Sunday 3:00 AM, keeps 4 weeks
0 3 * * 0 sqlite3 /home/mitkatch/hvac-engine/engine.db \
  ".backup /home/mitkatch/hvac-engine/backups/engine-$(date +\%Y\%m\%d).db" && \
  find /home/mitkatch/hvac-engine/backups -name "*.db" -mtime +28 -delete
Create the backups directory first: mkdir -p ~/hvac-engine/backups
